# Technology Stack

**Analysis Date:** 2026-03-18

## Languages

**Primary:**
- C - Device drivers and core embedded functionality
- Devicetree Source (DTS) - Hardware configuration and pin mapping
- Kconfig - Build configuration system
- C Preprocessor (DTS bindings) - Hardware abstraction

**Secondary:**
- YAML - Build configuration and GitHub Actions
- JSON - Layout definitions and metadata

## Runtime

**Environment:**
- Zephyr RTOS - Real-time operating system for embedded systems

**Package Manager:**
- west - Zephyr package and project manager
- Git - Project versioning and dependency management

## Frameworks

**Core:**
- ZMK (Zephyr Mechanical Keyboard) - Custom keyboard firmware framework built on Zephyr
  - Location: `config/`
  - Purpose: Keyboard matrix scanning, BLE connectivity, key binding system, mouse/trackball input handling

**Hardware Abstraction:**
- Nordic nRF Zephyr Support - Nordic nRF52 series MCU drivers
- Devicetree Bindings - Hardware description and driver instantiation

**Connectivity:**
- Bluetooth Low Energy (BLE) - Wireless keyboard-to-host communication
- USB UART - Serial communication for monitoring and configuration

## Key Dependencies

**Critical:**
- zmk - Main ZMK firmware framework
  - Source: `https://github.com/zmkfirmware/zmk`
  - Location: `config/west.yml` line 8-11
  - Purpose: Provides keyboard layer system, key bindings, BLE driver stubs, input handling

- zmk-pmw3610-driver - Pixart PMW3610 trackball/motion sensor driver
  - Source: `https://github.com/badjeff/zmk-pmw3610-driver`
  - Location: `config/west.yml` line 12-14
  - Purpose: SPI-based trackball input processing for Charybdis hardware

## Hardware Configuration

**MCU:**
- Nice!Nano V2 - nRF52840-based microcontroller board
  - Board: `nice_nano`
  - Supports: BLE, USB, GPIO, SPI, UART
  - Flash: 1MB
  - RAM: 256KB

**Peripherals:**
- PMW3610 Trackball Sensor - SPI optical motion tracker
  - Pins: SPI0 (SCK/MOSI/MISO) + IRQ pin
  - Configuration: `config/boards/shields/charybdis/pmw3610.dtsi`
  - CPI: 800 (configurable)
  - Input modes: Relative XY coordinates, scroll wheel

- Pro Micro GPIO Matrix
  - Row pins: GPIO 18, 5, 4, 9 (4 rows)
  - Column pins: GPIO 19, 20, 10, 6, 7, 8 (6 columns per half)
  - Diode direction: Row-to-column

## Build System

**Build Configuration:**
- CMake - Generated by ZMK from Kconfig and Devicetree
- Zephyr build scripts - Handles compilation, linking, and firmware generation

**Build Artifacts:**
- Firmware format: `.uf2` (UF2 bootloader format) for Nice!Nano
- Build output location: Generated during CI/CD pipeline
- Configuration: `config/boards/shields/charybdis/Kconfig.shield`

**Build Flags:**
- `CONFIG_ZMK_POINTING=y` - Enable pointer/mouse support
- `CONFIG_ZMK_STUDIO=y` - Enable ZMK Studio RPC over USB
- `CONFIG_ZMK_BLE_EXPERIMENTAL_CONN=y` - Experimental connection improvements
- `CONFIG_BT_CTLR_TX_PWR_PLUS_8=y` - Maximum BLE transmission power
- `CONFIG_ZMK_SPLIT_BLE_CENTRAL_BATTERY_LEVEL_FETCHING=y` - Fetch peripheral battery levels

## Configuration

**Environment:**
- Build triggered by GitHub Actions on push/PR
- CI/CD: `.github/workflows/build.yml`
- Uses ZMK shared build workflow: `zmkfirmware/zmk/.github/workflows/build-user-config.yml@main`

**Key Configuration Files:**
- `config/west.yml` - Project manifest and dependency sources
- `config/charybdis.conf` - Global ZMK configuration settings
- `config/charybdis.keymap` - Key binding definitions with layers and combos
- `build.yaml` - GitHub Actions matrix definition (board/shield combinations)

**Runtime Configuration:**
- Tapping term: 200ms (key hold timing)
- BLE connection parameters: Lower latency (9ms preferred interval, 16x latency)
- Behavior queue size: 512 events
- Battery monitoring: Enabled for split keyboard remote percentage
- Mouse scroll scaling: 1:20 ratio for trackball

## Platform Requirements

**Development:**
- Git and west package manager
- Python 3.8+ (for Zephyr tools)
- ARM GNU Toolchain (arm-none-eabi-gcc)
- nRF5 SDK or compatible development environment
- ZMK repository clone

**Production (Flash Target):**
- Nice!Nano V2 microcontroller board
- Charybdis mechanical keyboard shield (left/right variants)
- nRF52840 firmware programmer/bootloader support
- USB serial interface for configuration

**Deployment:**
- GitHub Actions CI builds `.uf2` firmware files automatically
- UF2 bootloader on Nice!Nano accepts drag-and-drop firmware updates
- Artifacts stored and retrievable from GitHub Actions
